# Muliao / 幕僚

Muliao 是一个基于 OpenClaw 等多 Agent 框架的「AI 助手团队」项目，目标是让普通人也能拥有一支长期在线、能真正“动手办事”的个人/工作 AI 团队。  

## 核心愿景

- 把大模型变成可靠的“执行者”，而不是只会聊天的问答机器人。  
- 让非技术用户也能通过熟悉的聊天入口（如 IM、协作工具）使用一整支 AI 助手团队。  
- 在安全可控的前提下，让助手持续学习用户偏好，逐步接管重复性事务。

## 设计理念

- **场景优先**：从具体生活和工作场景出发（个人效率、学习科研、家庭生活与理财），而不是从技能列表出发。  
- **幕僚团队**：团队长 + 专职幕僚——各有名字、性格和专长。用户可以直接找专人对话，也可以通过团队长统一交互。  
- **贴近日常的入口**：优先通过 WhatsApp / Telegram / Slack / 飞书等渠道接入，降低使用门槛。  
- **强安全感与可控感**：遵循最小权限原则，所有关键操作可追踪、可解释、可撤销。  
- **透明可编辑的记忆**：长期记忆和偏好配置以文档形式暴露，用户可以随时查看和修改。

## 能力概览

Muliao 基于 OpenClaw 提供的特性，重点构建以下能力层：

- 多渠道统一网关：一个后端接入多个聊天/协作工具，对话统一路由到相应 Agent。  
- 多 Agent 协作：支持“主助手 + 子代理”模式，实现需求拆解、分工与汇总。  
- 工作区与状态管理：为每个 Agent 提供独立 workspace（角色定义、记忆、技能配置等）。  
- 主动任务（心跳与定时）：支持“夜间整理邮件”“每周生成周报”等主动任务。  
- 可扩展技能体系：通过插件/Skills 形式引入社区和自定义能力（邮件、日历、文档、财务等）。

## 面向普通用户的目标

Muliao 的目标用户是非技术背景的个人和小团队，我们希望做到：

- 用自然语言就能完成初始化与配置，由向导式对话引导完成账号绑定和权限授权。  
- 提供一键启用的「场景模板」，例如：  
  - 个人效率助手（整理邮件、日程、待办）  
  - 学习 / 科研资料助手（检索、归档、生成摘要）  
  - 家庭生活与理财助手（账单整理、收支记录、提醒）  
- 每个模板清晰标注：能做什么、不做什么、需要哪些权限。

## 当前路线（Roadmap 简版）

1. MVP 场景落地  
   - 选择 1–2 个高频、低风险场景（如个人效率助手）实现第一版多 Agent 工作流。  

2. 非技术用户友好的控制面  
   - 提供 Web 控制台或聊天式“设置向导”，包装底层配置。  

3. 多 Agent 模板沉淀  
   - 抽象出可复用的角色与协作模式，形成可一键导入的配置模板。  

4. 体验评估与迭代  
   - 以时间节省、任务完成率、用户安心度等指标持续优化方案。  

5. 开放与社区  
   - 开源角色设计、模板和最佳实践，鼓励二次定制和共享，逐步形成 Muliao 的规范与生态。

## 适合谁使用

- 想把日常邮件、日程、文档整理交给 AI 的个人。  
- 希望为团队快速搭建一个“AI 助手团队”的小型公司或工作室。  
- 想基于 Muliao 开源中间件做垂直场景 AI 助手产品的下游公司。  
- 想基于 OpenClaw 做产品化探索的开发者和设计者。

## 商业模式

Muliao 采用三层互补的商业架构：开源中间件（生态基础）→ 自养机制（项目可持续）→ 母公司纵深应用（核心收入）。详见 [docs/business-model.md](docs/business-model.md)。
